SC1.py - SHOP SELLER/CLIENT + ENTRY COUNT + PEC (INTERACTION) COUNT
==================================================================
Standalone (no other project file needed). Author: Ahmed.

RUN
---
1) pip install -r requirementsVF.txt   (see the torch/CUDA note in that file)
2) In SC1.py set SOURCE_VIDEO_PATH (input) and TARGET_VIDEO_PATH (output).
3) python SC1.py
Weights (yolo11m.pt, osnet_ain_x1_0_msmt17.pt) auto-download on first run.
Output: an annotated video + printed totals (Clients entered, PEC events,
average interaction seconds).


WHAT IT DOES (per processed frame, every SKIP-th frame)
-------------------------------------------------------
1) DETECT people with YOLO11m.
2) DROP false detections: MIRROR_ZONES (reflections) and IGNORE_ZONES (static
   objects YOLO mis-detects, e.g. a handbag) -- by FEET-in-polygon.
3) TRACK with BoT-SORT (+OSNet ReID) -> a track id per detection. The id is used
   ONLY internally (the role is NOT decided by ReID). No ids are drawn.
4) SELLER vs CLIENT:
   - SEED: everyone in the first SEED_SECONDS is a SELLER (bootstraps staff).
   - UNIFORM: a person is SELLER if their lower body matches the BEIGE pants band
     AND they show a WHITE top. This VOTES per track: after SELLER_VOTES matching
     frames the track LOCKS as SELLER and stays SELLER on every later frame
     (anti-flicker), even when the uniform isn't visible -- as long as BoT-SORT
     keeps the same id.
   - SELLER ZONE: if >= SELLER_ZONE_FRAC of a person's BOX is inside a SELLER_ZONE
     they are SELLER (covers staff whose colour isn't readable; box-overlap, not
     feet, so a client whose feet only clip the zone is NOT made staff).
   - Everyone else = CLIENT.
5) CLIENT ENTRY COUNT (ENTREE-ZONE method): a CLIENT is counted ONCE when ALL hold:
     (1) its FEET (bottom-center) are inside ENTREE_ZONE,
     (2) its track id was not counted before,
     (3) it is a CLIENT,
     (4) it is moving INTO the shop (its signed distance to LINE1 changed inward
         since the previous frame by >= ENTER_MIN_STEP px).
   LINE1 is used ONLY to define which way is "inward" -- it is NOT a crossing line.
6) PEC (interaction): ONE engagement at a time, anchored to the SERVED client's
   POSITION. Opens when a client is near any seller; counted after PEC_MIN_SECS
   of continuous contact; ends when that position is gone for PEC_GRACE_SECS.
   A bystander passing behind does not split it; id churn mid-chat stays ONE PEC.


THE UNIFORM COLOR
-----------------
Uniform = BEIGE PANTS band (lower body) + WHITE TOP (upper body).
- PANTS: an HSV band. Source controlled by USE_REF_COLORS:
    USE_REF_COLORS = False -> use the fixed band UNIFORM_BAND_LO / UNIFORM_BAND_HI
                              baked in the code (no images needed at runtime).
    USE_REF_COLORS = True  -> re-learn the band from crops in SELLER_REF_DIR (+ the
                              first-3s seed); the run prints the learned band as
                              "[CALIB] uniform beige band H[..] S[..] V[..]" -- copy
                              it into UNIFORM_BAND_* and set USE_REF_COLORS=False.
- TOP: a fixed WHITE gate (pixels with S <= WHITE_S_MAX and V >= WHITE_V_MIN);
  white has no meaningful hue, so it is a low-saturation/bright rule, not a band.
A detection is "uniform" iff color_ratio(pants) >= COLOR_TAKE AND white_ratio >= WHITE_MIN.


ZONES (mark with a polygon picker; coords are for THIS camera)
--------------------------------------------------------------
- LINE1_START / LINE1_END : reference that defines "inward" for the entry count
                            (NOT a crossing line).
- ENTREE_ZONE   : the doorway. A client's FEET inside + moving inward => counted.
- SELLER_ZONES  : staff area. >= SELLER_ZONE_FRAC of the BOX inside => SELLER.
- MIRROR_ZONES  : drop reflected people (FEET in polygon).
- IGNORE_ZONES  : drop static false detections, e.g. a handbag (FEET in polygon). Keep TIGHT.


IMPORTANT PARAMETERS TO TUNE
----------------------------
SELLER vs CLIENT:
  COLOR_TAKE   (0.30) fraction of lower-body pixels inside the beige band to be "uniform".
               Lower = catch more sellers (more client false-positives); higher = stricter.
  WHITE_MIN    (0.25) white-top ratio required.
  SELLER_VOTES (2)    uniform frames before a track LOCKS as SELLER. 1 = instant
               (risk a client flicker locks); higher = slower but safer.
  SELLER_ZONE_FRAC (0.50) fraction of the BOX that must be inside a SELLER zone.
               NOTE: draw SELLER_ZONES over where the staff BODY is, not just floor.
  UNIFORM_BAND_LO/HI  the baked beige pants band (re-learn with USE_REF_COLORS to change).

ENTRY COUNTING (entree zone):
  ENTREE_ZONE     the doorway polygon (FEET inside).
  LINE1_START/END defines "inward" direction only.
  ENTER_SIGN      (+1) +1: inward = signed-dist increases; flip to -1 if it counts backwards.
  ENTER_MIN_STEP  (1)  min inward movement (px) per processed frame to count as "forward".
                       raise to reject doorway loitering/jitter; lower to catch slow steps.
  CONF_CLIENT     (0.2) min YOLO confidence for a client to be kept/counted.

PEC (interaction):
  PEC_PROX_DIST   (350) px gap to a seller to be "at the counter".
  PEC_MIN_SECS    (8)   continuous seconds before an interaction counts.
  PEC_GRACE_SECS  (12)  how long the served client may vanish before it ends.
  PEC_FOLLOW_DIST (160) how far the served client may drift and stay "the same".

GENERAL:
  DETECT_CONF (0.1) YOLO detection conf (low, to catch faint sellers).
  SKIP        (3)   process every 3rd frame (speed vs accuracy).
  DEBUG_CALIB (True) print the learned band + per-track stats at the end.
  MAX_FRAMES  (0)   >0 stops early for a quick test.


KNOWN LIMITATIONS
-----------------
- A seller can briefly flip to CLIENT if BoT-SORT changes their track id AND the
  uniform isn't visible at that instant (they must re-vote SELLER_VOTES frames, or
  be inside a SELLER_ZONE). Seed + vote-lock + seller-zone cover most cases, not all.
- The ENTRY count dedups by track id (condition 2); if an id churns at the doorway
  the same person could be counted twice (or a re-entry counted again).
- Colour separation is weak when the uniform colours are COMMON (e.g. white shirt +
  black pants); on such footage lean on the SELLER_ZONE + seed rather than colour.
